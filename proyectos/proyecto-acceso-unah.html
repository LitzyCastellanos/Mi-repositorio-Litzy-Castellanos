Descripción general del sistema

API REST construida con TypeScript y Node.js como proyecto de Ingeniería de Software para la UNAH. Expone endpoints para gestionar el ingreso y registro de personas al campus universitario: estudiantes, empleados y visitas (proveedores, contratistas).

Las reglas de negocio —como el control del formulario 03, contratos vigentes y el límite de vehículos por persona— se ejecutan directamente en PostgreSQL, manteniendo la lógica centralizada en la base de datos. El frontend fue desarrollado y desplegado desde v0.app con sincronización automática a Vercel.

"Sincronizado automáticamente con tus implementaciones de v0.app. Cualquier cambio en la aplicación desplegada se envía automáticamente a este repositorio."

Lógica del programa

  1. Enrutamiento versioned: todos los endpoints viven bajo /api/v1/ para soportar evolución sin romper clientes existentes.
  2. Upsert por DNI: los endpoints de personas hacen insert o update según si el DNI ya existe, evitando duplicados.
  3. Reglas de negocio en SQL: validaciones como formulario 03, contratos vigentes y límite de 2 vehículos activos se ejecutan con stored procedures o constraints en PostgreSQL.
  4. Control de vehículos: cada persona puede tener máximo 2 vehículos activos; el campo prestamo_temporal marca si es de uso temporal.
  5. Errores estandarizados: validaciones devuelven HTTP 400 con mensaje descriptivo; errores de negocio retornan 422.

Cómo funciona internamente

  • Runtime: Node.js 18+ con TypeScript compilado, tipado estricto en modelos y controladores.
  • Base de datos: PostgreSQL en localhost, BD sistema_ingreso, usuario app_rw.
  • Queries: SQL parametrizado directo para mantener la lógica de negocio en el motor de BD.
  • Frontend: generado y desplegado desde v0.app (React + CSS), sincronizado automáticamente con Vercel vía GitHub.
  • Variables de entorno: configuración de conexión centralizada en .env (no se incluye en el repo).

Endpoints de la API

Método Ruta Descripción
GET /health Estado del servidor
GET /api/v1/personas/:dni Buscar persona por DNI
POST /api/v1/personas/estudiante Crear o editar estudiante
POST /api/v1/personas/empleado Crear o editar empleado
POST /api/v1/visitas Crear o editar visita/proveedor
POST /api/v1/vehiculos Registrar vehículo (máx. 2 activos)

Guía de instalación paso a paso

Requisitos previos

  • Node.js 18 o superior
  • PostgreSQL corriendo en localhost con la BD sistema_ingreso y usuario app_rw
  • Git instalado
  • Terminal / línea de comandos
  1. Clonar el repositorio
    git clone https://github.com/LitzyCastellanos/v0-university-login-system.git
    cd v0-university-login-system
  2. Copiar y ajustar variables de entorno
    cp .env.example .env
    Editar .env con las credenciales de tu instancia PostgreSQL.
  3. Instalar dependencias
    npm install
  4. Levantar el servidor en desarrollo
    npm run dev
    El servidor queda disponible en http://localhost:3000

Notas importantes

  • Los errores de validación devuelven HTTP 400 con mensaje descriptivo.
  • Las reglas de negocio (formulario 03, contratos, límite de vehículos) se ejecutan en SQL directamente en PostgreSQL.
  • El campo face_template_b64 acepta un template facial en base64 o null si no se utiliza reconocimiento facial.
  • El límite de 2 vehículos activos por persona es validado a nivel de base de datos.
